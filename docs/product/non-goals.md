# Non-Goals

## Purpose

This document protects the project from uncontrolled scope expansion.

DRIFT PURSUIT GRID aims to be deep, polished, deterministic, and explainable.

It does not aim to become everything.

## Non-Goal 1 — Full Vehicle Physics Simulation

The project does not attempt to simulate real-world vehicle physics exhaustively.

The goal is simplified, expressive, readable vehicle dynamics.

## Non-Goal 2 — 3D Rendering

The project is terminal-native by design.

The terminal is not a limitation. It is part of the product identity.

## Non-Goal 3 — Open World Simulation

The project does not need a large open world.

It focuses on structured urban pursuit scenarios.

## Non-Goal 4 — Multiplayer Networking

The first-class experience is local, deterministic, replayable simulation.

Networking would increase complexity and weaken the first version.

## Non-Goal 5 — Random Chaos

The project does not use randomness as a replacement for design.

Randomness must be seeded, controlled, explainable, and replayable.

## Non-Goal 6 — HUD Overload

The project must not bury the player under excessive information.

The HUD should be minimal, readable, and useful.

## Non-Goal 7 — Seriousness That Kills Fun

The assurance model must support the gameplay.

It must never make the game feel slow, cold, or academic.
